## Positioning statement

Generic assistants answer prompts. Michael Agent maintains governed state, handles evidence, pressure-tests claims, plans under authority, and emits an audit trail. Prophet Mesh is the runtime fabric that gives Michael identity, memory, tools, lifecycle, policy, revocation, and deployment topology.

## Product pillars

### Evidence-native reasoning

Michael tracks claims, sources, contradictions, confidence, and open questions. Material reasoning should be inspectable, not ephemeral.

### Governed action planning

Plans name authority, motive, required evidence, policy gates, and revocation paths before execution.

### Durable memory

Prophet Mesh connects Michael to scoped memory graphs so work compounds without collapsing customer data boundaries.

### Premium customization

Customers can derive domain-specific agents from Michael: custom vocabulary, connectors, workflows, UI surface, deployment topology, and policy overlays.

### Trust kernel

Identity, policy, evidence, attestation, revocation, and audit are non-negotiable runtime properties.

Claude-style general intelligence is the market reference point. Prophet Mesh competes by making the agent operationally governable: evidence, lifecycle, policy, memory boundaries, premium customization, and attestable execution are designed into the product instead of bolted on later.
